Alur manajemen yang disarankan

1. Upload ZIP

Taruh file ke docs/downloads/plugin-id/, atau pakai GitHub Releases kalau ukuran besar.

2. Update manifest

Edit docs/data/plugins.json: latestVersion, stableVersion, favoriteVersion, dan versions[].

3. Test lalu tandai

Latest boleh candidate/testing. Stable hanya untuk versi yang sudah aman di project.

4. Arsipkan versi lama

Jangan hapus versi lama. Ganti status menjadi deprecated/archived agar riwayat tetap jelas.
